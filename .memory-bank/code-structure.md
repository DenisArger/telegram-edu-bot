# Детальная структура кода

## src/config.js

**Назначение**: Загрузка и валидация переменных окружения

**Экспорты**:
- `config` - объект с конфигурацией:
  - `telegramToken` - токен Telegram бота
  - `airtableApiKey` - API ключ Airtable
  - `airtableBaseId` - ID базы данных Airtable

**Функции**:
- `validateConfig()` - проверяет наличие всех обязательных переменных, выбрасывает ошибку при отсутствии

**Особенности**:

- Использует `dotenv` для загрузки `.env` файла

## src/airtable.js

**Назначение**: Абстракция для работы с Airtable API

**Экспорты**:
- `getRecords(tableName, filterFormula)` - получение записей
- `createRecord(tableName, fields)` - создание записи
- `updateRecord(tableName, recordId, fields)` - обновление записи

**Детали реализации**:

### getRecords
```javascript
getRecords(tableName, filterFormula = "")
```
- Получает все записи из указанной таблицы
- Поддерживает фильтрацию через Airtable formula
- Обрабатывает пагинацию автоматически через `eachPage`
- Возвращает массив Airtable records

### createRecord
```javascript
createRecord(tableName, fields)
```
- Создает новую запись в указанной таблице
- `fields` - объект с полями записи
- Возвращает созданную запись

### updateRecord
```javascript
updateRecord(tableName, recordId, fields)
```
- Обновляет существующую запись
- `recordId` - ID записи в Airtable
- `fields` - объект с полями для обновления
- Возвращает обновленную запись

**Обработка ошибок**:
- Все функции логируют ошибки в консоль
- Пробрасывают ошибки наверх для обработки вызывающим кодом

## src/student.js

**Назначение**: Работа с данными учеников

**Экспорты**:
- `getStudentByTelegramId(telegramId)`
- `createStudent(telegramId, lessonId, name)`
- `getAvailableLessons()`
- `updateStudentLesson(studentId, lessonId)`

**Детали реализации**:

### getStudentByTelegramId
```javascript
getStudentByTelegramId(telegramId: number): Promise<Object|null>
```
- Ищет ученика по Telegram ID в таблице "Ученики"
- Использует фильтр Airtable: `{Telegram ID} = ${telegramId}`
- Возвращает объект с `id` и `fields`, или `null` если не найден
- Нормализует формат записи для удобства использования

### createStudent
```javascript
createStudent(telegramId: number, lessonId: string, name?: string): Promise<Object>
```
- Создает новую запись ученика в Airtable
- Устанавливает Telegram ID и текущий урок
- Опционально добавляет имя, если указано
- Возвращает созданную запись в нормализованном формате

### getAvailableLessons
```javascript
getAvailableLessons(): Promise<Array<{id: string, name: string}>>
```
- Получает все уроки из таблицы "Уроки"
- Возвращает массив объектов с `id` и `name`
- Поддерживает поля "Название" или "Имя" для названия урока

### updateStudentLesson
```javascript
updateStudentLesson(studentId: string, lessonId: string): Promise<Object>
```
- Обновляет текущий урок ученика
- Использует Link field (массив) для связи с уроком
- Возвращает обновленную запись

## src/tasks.js

**Назначение**: Получение и проверка заданий

**Экспорты**:
- `getTasksForLesson(lessonId, studentClass)`
- `checkAnswer(userInput, answers)`

**Внутренние функции**:
- `shuffleArray(array)` - перемешивание массива (Fisher-Yates)

**Детали реализации**:

### getTasksForLesson
```javascript
getTasksForLesson(lessonId: string, studentClass?: string|number|null): Promise<Array>
```
**Логика работы**:
1. Получает все активные задания (`{Активно} = TRUE()`)
2. Фильтрует задания по уроку (проверяет Link field "Урок")
3. Фильтрует задания по классу (если указан):
   - Если у ученика есть класс: показывает задания без класса + задания для его класса
   - Если у ученика нет класса: показывает только задания без класса
4. Преобразует записи в объекты заданий:
   - `id` - ID записи
   - `text` - текст задания
   - `answers` - массив правильных ответов (нормализованных)
   - `correctFeedback` - комментарий при правильном ответе
   - `wrongFeedback` - комментарий при неправильном ответе
   - `order` - порядок (не используется, т.к. задания перемешиваются)
5. Перемешивает задания в случайном порядке
6. Возвращает массив заданий

**Особенности фильтрации по классу**:
- Поддерживает как строковые, так и числовые значения класса
- Нормализует значения (trim, toLowerCase для строк)
- Обрабатывает Link fields (массивы)
- Логирует процесс фильтрации для отладки

### checkAnswer
```javascript
checkAnswer(userInput: string, answers: Array<string>): boolean
```
- Нормализует входной ответ (trim, toLowerCase)
- Проверяет наличие в массиве правильных ответов
- Возвращает `true` если ответ правильный, иначе `false`
- Обрабатывает пустые входные данные

## src/bot.js

**Назначение**: Основной файл бота - обработка команд и сообщений

**Инициализация**:
```javascript
const bot = new TelegramBot(config.telegramToken, { polling: true });
```

**Управление состоянием**:
- `userState` (Map) - хранит состояние для каждого chatId
- `registrationState` (Map) - не используется в текущей версии

**Обработчики**:

### 1. Команда /start
```javascript
bot.onText(/\/start/, async (msg) => { ... })
```
**Логика**:
- Проверяет регистрацию ученика
- Если не зарегистрирован: предлагает регистрацию
- Если зарегистрирован: показывает главное меню с кнопками:
  - "Начать урок"
  - "Выбрать другой урок"
  - "Информация"

### 2. Команда /register
```javascript
bot.onText(/\/register/, async (msg) => { ... })
```
**Логика**:
- Проверяет, не зарегистрирован ли уже
- Получает список доступных уроков
- Показывает inline-клавиатуру с уроками
- При выборе урока создает запись ученика

### 3. Callback queries (кнопки)
```javascript
bot.on("callback_query", async (query) => { ... })
```

**Обрабатываемые callback_data**:
- `start_register` - начало регистрации
- `register_lesson_{lessonId}` - выбор урока при регистрации
- `start_lesson` - начало урока
- `change_lesson` - выбор другого урока
- `change_to_lesson_{lessonId}` - изменение на конкретный урок
- `skip_task_{index}` - пропуск задания
- `end_lesson` - завершение урока
- `restart_lesson` - перезапуск урока
- `main_menu` - возврат в главное меню
- `show_info` - показ информации

### 4. Обычные сообщения (ответы на задания)
```javascript
bot.on("message", async (msg) => { ... })
```
**Логика**:
- Пропускает команды (начинающиеся с `/`)
- Проверяет наличие активного состояния
- Получает текущее задание из состояния
- Проверяет ответ через `checkAnswer`
- Сохраняет прогресс через `saveProgress`
- Отправляет комментарий (правильный/неправильный)
- При правильном ответе переходит к следующему заданию
- При неправильном ответе позволяет попробовать снова

### 5. Обработка ошибок polling
```javascript
bot.on("polling_error", (error) => { ... })
```
- Обрабатывает конфликты (409 - другой экземпляр бота)
- Логирует ошибки
- Автоматический retry при конфликтах

**Вспомогательные функции**:

### sendFirstTask
```javascript
async function sendFirstTask(chatId, student)
```
**Логика**:
1. Получает ID текущего урока ученика
2. Получает класс ученика для фильтрации
3. Загружает задания для урока с учетом класса
4. Инициализирует состояние пользователя
5. Отправляет первое задание с кнопками управления

**Особенности**:
- Обрабатывает отсутствие урока
- Обрабатывает отсутствие заданий
- Логирует процесс для отладки фильтрации по классу

## Особенности реализации

### Обработка Link fields в Airtable
- Link fields возвращаются как массивы ID
- Код проверяет, является ли значение массивом
- При сравнении используется `includes()` для массивов

### Нормализация данных
- Все ответы нормализуются (trim, toLowerCase)
- Классы нормализуются (trim, приведение к строке)
- Обработка пустых значений и null

### Управление состоянием
- Состояния хранятся в памяти (Map)
- При перезапуске бота состояния теряются
- Ученик может начать урок заново

### Обработка ошибок
- Все async функции обернуты в try-catch
- Ошибки логируются в консоль
- Пользователю отправляются понятные сообщения об ошибках
- Ошибки сохранения прогресса не прерывают выполнение урока
