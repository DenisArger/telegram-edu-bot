# Обзор проекта: Telegram Bot для школьных уроков

## Краткое описание

Telegram-бот для выдачи заданий по школьным предметам с интеграцией Airtable. Бот предназначен для учеников и работает исключительно через Telegram. Учитель управляет данными через Airtable без необходимости работы с кодом.

## Основная информация

- **Название проекта**: telegram-edu-bot
- **Тип**: Telegram Bot (Node.js)
- **Версия**: 1.0.0
- **Язык программирования**: JavaScript (ES Modules)
- **Платформа**: Node.js ≥ 18

## Основные возможности

1. **Выдача заданий по школьным предметам**
   - Задания загружаются из Airtable
   - Поддержка фильтрации по классу ученика
   - Случайный порядок заданий
   - Универсальная система для любых предметов

2. **Проверка ответов учеников**
   - Автоматическая проверка правильности ответов
   - Поддержка множественных правильных ответов
   - Нормализация ответов (регистр, пробелы)

3. **Сохранение прогресса в Airtable**
   - Каждый ответ сохраняется в таблице "Прогресс"
   - Отслеживание правильности ответов
   - История выполнения заданий

4. **Комментарии к ответам**
   - Персонализированные комментарии при правильном ответе
   - Комментарии при неправильном ответе
   - Настраиваемые через Airtable

5. **Управление уроками через Airtable**
   - Учитель может создавать/редактировать уроки без изменения кода
   - Управление заданиями через интерфейс Airtable
   - Назначение уроков ученикам

6. **Регистрация учеников**
   - Автоматическая регистрация через бота
   - Выбор урока при регистрации
   - Возможность смены урока

## Технологический стек

- **Node.js** - среда выполнения
- **node-telegram-bot-api** - библиотека для работы с Telegram Bot API
- **airtable** - официальная библиотека для работы с Airtable API
- **dotenv** - управление переменными окружения

## Структура базы данных Airtable

### Таблица "Ученики"
- `Telegram ID` (Number) - ID пользователя в Telegram
- `Текущий урок` (Link to "Уроки") - связь с текущим уроком
- `Класс` (Text/Number) - класс ученика (опционально, для фильтрации заданий)

### Таблица "Уроки"
- `Название` или `Имя` (Single line text) - название урока

### Таблица "Задания"
- `Активно` (Checkbox) - активность задания
- `Урок` (Link to "Уроки") - связь с уроком
- `Текст задания` (Single line text) - текст задания
- `Правильные ответы` (Long text) - правильные ответы (каждый с новой строки)
- `Комментарий (верно)` (Single line text) - комментарий при правильном ответе
- `Комментарий (ошибка)` (Single line text) - комментарий при неправильном ответе
- `order` или `Порядок` (Number) - порядок задания в уроке (не используется, задания показываются в случайном порядке)
- `Класс` (Text/Number) - класс, для которого предназначено задание (опционально)

### Таблица "Прогресс"
- `Ученик` (Link to "Ученики") - связь с учеником
- `Задание` (Link to "Задания") - связь с заданием
- `Ответ ученика` (Single line text) - ответ ученика
- `Верно` (Checkbox) - правильность ответа

## Команды бота

- `/start` - Главное меню (для зарегистрированных) или предложение регистрации
- `/register` - Регистрация нового ученика с выбором урока

## Пользовательский интерфейс

Бот использует inline-клавиатуры для навигации:
- Кнопки для выбора действий
- Кнопки для выбора уроков
- Кнопки управления уроком (пропустить задание, завершить урок)
- Кнопки навигации (главное меню, выбор урока)

## Состояния пользователя

Бот управляет состояниями пользователей через Map:
- `userState` - хранит текущий индекс задания и список заданий для каждого пользователя
- `registrationState` - хранит состояние процесса регистрации (не используется в текущей версии)

## Обработка ошибок

- Валидация переменных окружения при запуске
- Обработка ошибок API Airtable
- Обработка ошибок Telegram API
- Информативные сообщения об ошибках для пользователей
- Обработка конфликтов при запуске нескольких экземпляров бота

## Безопасность

- Переменные окружения хранятся в `.env` (не коммитится в Git)
- `.env.example` содержит только placeholder значения
- Использование Personal Access Token для Airtable
- Валидация всех входных данных
