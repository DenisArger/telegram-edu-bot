# Архитектура проекта

## Структура файлов

```
telegram-edu-bot/
├── src/
│   ├── bot.js              # Основной файл бота - обработчики команд и сообщений
│   ├── airtable.js         # Модуль для работы с Airtable API
│   ├── student.js          # Функции для работы с данными учеников
│   ├── tasks.js            # Получение и проверка заданий
│   ├── progress.js         # Сохранение прогресса в Airtable
│   ├── config.js           # Конфигурация и загрузка переменных окружения
│   ├── check-tasks.js      # Утилита для проверки заданий (тестовый скрипт)
│   ├── test-airtable.js    # Тестовый скрипт для проверки подключения к Airtable
│   └── test-task-filter.js # Тестовый скрипт для проверки фильтрации заданий
├── .memory-bank/           # Документация проекта
├── package.json            # Зависимости и скрипты проекта
├── Procfile               # Конфигурация для Heroku/Railway
├── yarn.lock              # Зафиксированные версии зависимостей
└── README.md              # Основная документация
```

## Модульная архитектура

### 1. config.js
**Назначение**: Загрузка и валидация переменных окружения

**Функции**:
- Загружает переменные из `.env` через `dotenv`
- Экспортирует объект `config` с токенами и ключами
- Валидирует наличие всех обязательных переменных при загрузке модуля

**Зависимости**: `dotenv`

### 2. airtable.js
**Назначение**: Абстракция для работы с Airtable API

**Функции**:
- `getRecords(tableName, filterFormula)` - получение записей из таблицы
- `createRecord(tableName, fields)` - создание новой записи
- `updateRecord(tableName, recordId, fields)` - обновление записи

**Особенности**:
- Использует официальную библиотеку `airtable`
- Обрабатывает пагинацию автоматически
- Обрабатывает ошибки и логирует их

**Зависимости**: `airtable`, `config.js`

### 3. student.js
**Назначение**: Работа с данными учеников

**Функции**:
- `getStudentByTelegramId(telegramId)` - получение ученика по Telegram ID
- `createStudent(telegramId, lessonId, name)` - создание нового ученика
- `getAvailableLessons()` - получение списка доступных уроков
- `updateStudentLesson(studentId, lessonId)` - обновление текущего урока ученика

**Зависимости**: `airtable.js`

### 4. tasks.js
**Назначение**: Получение и проверка заданий

**Функции**:
- `getTasksForLesson(lessonId, studentClass)` - получение заданий для урока
- `checkAnswer(userInput, answers)` - проверка правильности ответа
- `shuffleArray(array)` - перемешивание массива (внутренняя функция)

**Особенности**:
- Фильтрация заданий по классу ученика
- Случайный порядок заданий (Fisher-Yates shuffle)
- Поддержка множественных правильных ответов
- Нормализация ответов (регистр, пробелы)

**Зависимости**: `airtable.js`

### 5. progress.js
**Назначение**: Сохранение прогресса выполнения заданий

**Функции**:
- `saveProgress(studentId, taskId, answer, isCorrect)` - сохранение ответа ученика

**Зависимости**: `airtable.js`

### 6. bot.js
**Назначение**: Основной файл бота - обработка команд и сообщений

**Основные компоненты**:
- Инициализация Telegram Bot с polling
- Управление состояниями пользователей (Map)
- Обработчики команд (`/start`, `/register`)
- Обработчики callback queries (кнопки)
- Обработчики обычных сообщений (ответы на задания)
- Обработка ошибок polling

**Потоки обработки**:

1. **Регистрация**:
   - `/register` или кнопка "Зарегистрироваться"
   - Получение списка уроков
   - Выбор урока через inline-клавиатуру
   - Создание записи ученика в Airtable

2. **Начало урока**:
   - `/start` → главное меню
   - Кнопка "Начать урок"
   - Загрузка заданий для текущего урока ученика
   - Фильтрация по классу (если указан)
   - Отправка первого задания

3. **Выполнение заданий**:
   - Получение ответа от пользователя
   - Проверка правильности
   - Сохранение прогресса
   - Отправка комментария
   - Переход к следующему заданию (если правильно)

4. **Управление уроком**:
   - Пропуск задания
   - Завершение урока
   - Выбор другого урока
   - Перезапуск урока

**Зависимости**: Все модули проекта

## Потоки данных

### Регистрация ученика
```
Telegram → bot.js → student.js → airtable.js → Airtable API
```

### Начало урока
```
Telegram → bot.js → student.js → airtable.js → Airtable API (получение ученика)
         → tasks.js → airtable.js → Airtable API (получение заданий)
         → Telegram (отправка задания)
```

### Проверка ответа
```
Telegram → bot.js → tasks.js (проверка)
         → progress.js → airtable.js → Airtable API (сохранение)
         → Telegram (комментарий)
```

## Управление состоянием

Бот использует in-memory хранилище состояний:
- `userState` (Map) - хранит для каждого chatId:
  - `taskIndex` - текущий индекс задания
  - `tasks` - массив заданий текущего урока
  - `studentId` - ID ученика в Airtable

**Особенности**:
- Состояния хранятся только в памяти
- При перезапуске бота состояния теряются
- Ученик может начать урок заново через `/start`

## Обработка ошибок

### Уровни обработки ошибок:

1. **Уровень модулей** (airtable.js, student.js, tasks.js):
   - Логирование ошибок в консоль
   - Проброс ошибок наверх

2. **Уровень бота** (bot.js):
   - Try-catch блоки в обработчиках
   - Информативные сообщения пользователю
   - Логирование ошибок для отладки

3. **Уровень polling**:
   - Обработка конфликтов (409 - другой экземпляр бота)
   - Автоматический retry при ошибках

## Масштабируемость

### Текущие ограничения:
- Состояния хранятся в памяти (не подходит для нескольких экземпляров)
- Нет очереди сообщений
- Нет кэширования данных из Airtable

### Возможные улучшения:
- Использование Redis для хранения состояний
- Кэширование заданий и уроков
- Webhook вместо polling для production
- База данных для хранения состояний

## Безопасность

- Валидация всех входных данных
- Нормализация ответов перед проверкой
- Защита от SQL injection (Airtable API обрабатывает это)
- Переменные окружения для секретов
- Проверка существования ученика перед операциями
