# Memory Bank - Документация проекта

Этот каталог содержит структурированную документацию всего проекта Telegram-бота для школьных уроков.

## Структура документации

### 📋 project-overview.md
**Общий обзор проекта**
- Краткое описание и назначение
- Основные возможности
- Технологический стек
- Структура базы данных Airtable
- Команды бота и пользовательский интерфейс

### 🏗️ architecture.md
**Архитектура проекта**
- Структура файлов
- Модульная архитектура с описанием каждого модуля
- Потоки данных
- Управление состоянием
- Обработка ошибок
- Масштабируемость

### 📚 documentation-index.md
**Индекс всей документации проекта**
- Описание всех .md файлов в проекте
- Быстрая навигация по документации
- Рекомендации, с чего начать

### 💻 code-structure.md
**Детальная структура кода**
- Описание каждого файла исходного кода
- Детали реализации функций
- Примеры использования
- Особенности обработки данных

### 🚀 setup-and-deployment.md
**Руководство по настройке и деплою**
- Локальная настройка проекта
- Настройка Airtable
- Деплой на различные платформы (Railway, Render, Heroku, VPS)
- Решение проблем
- Мониторинг и обновление

## Как использовать эту документацию

### Для новых разработчиков:
1. Начните с **project-overview.md** - получите общее понимание проекта
2. Изучите **architecture.md** - поймите структуру и архитектуру
3. Прочитайте **code-structure.md** - изучите детали реализации
4. Используйте **setup-and-deployment.md** - настройте локальное окружение

### Для быстрого поиска информации:
- Используйте **documentation-index.md** как навигатор
- Ищите нужную информацию по ключевым словам в файлах
- Следуйте ссылкам между документами

### Для деплоя:
- Следуйте инструкциям в **setup-and-deployment.md**
- Используйте информацию из **architecture.md** для понимания требований

## Связь с основной документацией

Эта документация дополняет основную документацию проекта:
- **README.md** - быстрый старт и основная информация
- **task.md** - исходное техническое задание
- Другие .md файлы в корне проекта - специализированные инструкции

## Обновление документации

При изменении проекта обновляйте соответствующую документацию:
- Изменения в архитектуре → **architecture.md**
- Новые функции → **project-overview.md** и **code-structure.md**
- Изменения в настройке → **setup-and-deployment.md**
- Новые документы → **documentation-index.md**
